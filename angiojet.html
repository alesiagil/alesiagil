About

Overview

Boston Scientific engaged our team to design the GUI for the AngioJet Platinum — a peripheral thrombectomy device that removes clots from arteries and veins via lytic delivery and active aspiration. Operated by a cardiovascular technologist in the OR and cath lab, while a surgeon directs.

What made this project demanding wasn't technical complexity — it was the stakes. A hesitation, a missed alert, or an ambiguous label at the wrong moment isn't just a UX problem. It's a patient safety issue.

The Challenge

The existing device: a small LCD and six hardware buttons. Techs ran procedures entirely from memory, carrying a sequence the interface did nothing to support. The fix wasn't more words and guidance — it was smarter structure.

Our job was to design an interface that thinks alongside the technologist, not one they have to think around.

From a 6-button LCD to a full guided experience

The AngioJet Ultra gave technologists a single text readout and a row of unlabeled icons. We designed a touchscreen system from the ground up — guided, phase-by-phase, with clear direction when something goes wrong.

Before — AngioJet Ultra

Small blue LCD, 6 status icons, segment-display timer, 3 physical buttons. No guided workflow, no touch interaction.

After — AngioJet Platinum

Full touchscreen GUI with guided procedure workflow, real-time data visualization, and role-specific displays.

03

Key Insights

Technologists don't want more controls — they want to be guided

The assumption was that techs needed a powerful, feature-rich interface. What we found was the opposite. They wanted a system that moved with them — one that said "you're here, this is what to do next" rather than "here are all the options, good luck." In a surgical setting, hesitation is a risk factor. The interface had to make the right action obvious without the user having to ask: what do I do right now?

Two users, one screen — with opposite needs

The tech manages the device — hands on, decisions fast, attention split between the screen and the surgeon's instructions. The surgeon monitors from across the room — glancing, interpreting, directing. Every screen had to serve precise hands-on control for the tech while remaining instantly readable from two feet away.

04

Aha! Moments

What if we designed the interface around the procedure itself — not the device?

Most medical device UIs organize around features. We organized ours around the procedure: setup → catheter priming → mode selection → active procedure → end. Each phase is a discrete screen state — the interface shows where you are and what to do next. Navigation wasn't about menus. It was about progression.

What do our two different users need to do, when and how?

The tech interacts up close. The surgeon reads from across the room. The hierarchy had to serve both simultaneously — no mode switching, no screen changes. This became the highest-impact design constraint, especially on the run screen.

05

Key Decisions

Narrow the options — deliberately

Each screen shows only what's relevant to the current phase — no jumping ahead, no skipping steps. This wasn't about limiting users. It was about protecting their attention in a high-stakes environment where decision fatigue is a real risk. A narrowed interface is a safer one.

Visual hierarchy as a clinical safety decision

The device's proprietary Power Pulse mode lets surgeons deliver a lytic agent before traditional aspiration — a powerful feature that also doubles the monitoring load and adds a new decision variable.

The run screen tracks 9 live data points, 7 of them timers. Making that legible without overwhelming either user was the core visual challenge.

The Interface

The guided setup flow

Step-by-step sequence with a clear active state and a hint of what's next. No way to skip ahead or miss a step. The interface does the remembering.

Setup & Priming

Mode selection

A flipping card reveals mode details before confirming. More fluid than a modal, less alarming. Users felt confident in the choice before committing.

Select Mode

Active procedure screens

7 timers on one screen — some always visible, some contextual. It shouldn't work. It does.

The main timer is patented: dual-progress, tracking per-mode limits and total procedure time simultaneously. Active mode is always clearly surfaced without obscuring the other.

Active Procedure

Dwell Timer

A collapsible drawer that tracks medicinal liquid time in the vein. Auto-tracking, with a reminder. That's it. The drawer collapses so procedure information stays in focus — feature-forward without the clutter.

Outcomes

The design cleared final-round usability testing with zero task errors across both user types.

Both user types completed every task correctly on first attempt — the FDA human factors benchmark. Biggest gains: error recovery speed and setup confidence. Technologists who'd run the old device for years said it felt "like it was designed for how the procedure actually works."

Delivered: 40+ hi-fi screens, a complete design system, and all documentation for the regulatory pathway.
